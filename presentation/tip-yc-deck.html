System 2 — Sell Audit Pipeline

The hardest problem in retail investing:
knowing when to exit.

Every holding is analyzed — not just flagged stocks. 5 parallel-enrichment stages, then Claude with extended thinking delivers a scored verdict.

5-Stage Pipeline
1
Portfolio Inspector — Live holdings + instrument tokens + sector lookup
2
Quant Analyst — RSI, ADX, EMA-20/50/200, ATR, 3M relative strength (parallel)
3
Fundamentals Analyst — Screener.in quarterly trends, ROE, D/E (parallel)
4
Sector Monitor — 5-day sector index performance vs each holding
5
Claude AI Ranker — Extended thinking, urgency 0–100, rationale
Why Claude for Exit Decisions

Exit analysis is fundamentally asymmetric — you need to reason across multiple conflicting signals simultaneously. Gemini handles the buy funnel (structured filters); Claude's extended thinking handles the nuanced, multi-dimensional exit reasoning where context matters more than rules.

Output — Urgency Scoring
HOLD 0–30 · Trend intact, fundamentals healthy, no exit signals
WATCH 31–60 · Mixed signals, some deterioration, monitor closely
SELL 61–80 · Clear deterioration, consider reducing position
STRONG SELL 81–100 · Structural breakdown, exit immediately
Automation also runs sell audit on Sunday. Any open automation position scoring ≥ 70 urgency is auto-closed before Monday's buy run.

Key Design Decisions
✓
Hybrid SL-M — broker-side stop placed immediately after fill. Protection exists even if our server goes down.
✓
Crash recovery — PENDING trades in SQLite are resumed by OrderTracker on server restart.
✓
Position reconciliation — compares DB state vs Kite holdings; flags untracked, marks external closes.
✓
Typed confirmation — switching to live requires user to type "I understand this uses real money".
Weekly Automation Engine

Monday 9AM IST. Fully autonomous.
No human required.

APScheduler fires the pipeline, runs multi-gear stock selection, deduplications across gears, checks VIX regime, executes trades, and logs everything — every week.

Weekly Run Sequence
Check NSE holiday calendar
Skip non-trading days automatically
Count open automation positions
Target 6 total. stocks_to_buy = 6 − open_count
Run sell audit — close weak positions
Auto-close if sell urgency ≥ 70
Run pipeline: Turbo → Balanced → Growth
Top 2 per gear, deduped across gears
Fallback: Cautious → Fortress if shortfall
Looks max 10 deep into each gear's ranked list
Execute trades + log run record
Persisted to automation_state.json + SQLite
VIX Regime Adaptation
Tier 1 — VIX 20–25 RSI tightened by −3
Tier 2 — VIX 25–30 RSI tightened by −7
Tier 3 — VIX >30 Pipeline skipped entirely
Position Sizing — ATR-Based
# Risk 1% of capital per trade risk_per_trade = capital × 0.01 risk_per_share = ATR × trail_multiplier quantity = risk_per_trade ÷ risk_per_share # Cap at 20% of capital per position max_qty = (capital × 0.20) ÷ ltp quantity = min(quantity, max_qty)
Drawdown Circuit Breaker

If account balance has dropped more than 15% from initial capital, the automation skips the entire buy run regardless of market conditions.
